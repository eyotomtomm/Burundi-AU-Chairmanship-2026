# Security Vulnerabilities Fixed - February 28, 2026

## Critical Security Issues Resolved

### 1. ❌ Hardcoded Localhost API URLs (CRITICAL)

**Problem:**
- API URL hardcoded to `http://localhost:8000/api` in production builds
- All users would see blank screens with no content
- HTTP (not HTTPS) sends all data including auth tokens in plaintext
- iOS App Transport Security blocks HTTP by default

**Files Affected:**
- `lib/config/app_constants.dart` (line 27)
- `lib/services/remote_config_service.dart` (line 35)
- `lib/services/api_service.dart` (line 15)

**Fix Applied:** ✅
- Created `lib/config/environment.dart` with environment-based configuration
- Supports Development, Staging, and Production environments
- Production uses HTTPS: `https://api.burundi4africa.com/api`
- Development uses localhost: `http://localhost:8000/api`
- Staging uses: `https://staging-api.burundi4africa.com/api`
- Build-time configuration via `--dart-define=ENVIRONMENT=production`

### 2. ❌ Hardcoded Media URL Workarounds (MAJOR)

**Problem:**
- Multiple screens used `.replaceAll('127.0.0.1', 'localhost')` workarounds
- Would fail in production when backend returns production URLs
- No centralized URL handling

**Files Affected:**
- `lib/screens/magazine/pdf_viewer_screen.dart` (line 77)
- `lib/screens/agenda/water_sanitation_screen.dart` (line 141)
- `lib/screens/agenda/arise_initiative_screen.dart` (line 139)
- `lib/screens/agenda/peace_security_screen.dart` (line 139)
- `lib/screens/home/home_screen.dart` (line 927)

**Fix Applied:** ✅
- Implemented `Environment.fixMediaUrl()` function
- Automatically handles localhost → production domain conversion
- Enforces HTTPS in production
- iOS-compatible (uses localhost instead of 127.0.0.1)

### 3. ❌ No Environment Separation (CRITICAL)

**Problem:**
- No way to build different versions for development vs production
- Same hardcoded URLs for all builds
- Cannot test against staging environment

**Fix Applied:** ✅
- Complete environment system with three tiers
- Build commands:
  - Development: `flutter build ios` (default)
  - Staging: `flutter build ios --dart-define=ENVIRONMENT=staging`
  - Production: `flutter build ios --dart-define=ENVIRONMENT=production`

## Backend Security - Additional Critical Fix (Feb 28, 2026)

### 🚨 SECRET_KEY Hardcoded Fallback Removed

**Problem:**
```python
# OLD CODE - VULNERABLE
SECRET_KEY = os.environ.get(
    'DJANGO_SECRET_KEY',
    'django-insecure-dev-key-ONLY-FOR-LOCAL-DEVELOPMENT',  # ❌ DANGEROUS
)
```

Even though this was "only for development", having ANY fallback value in version control is a critical vulnerability:
- Attackers can forge session cookies
- CSRF tokens can be generated by attackers
- JWT signatures can be forged
- If production accidentally runs without the env var, it uses the known key

**Fix Applied:**
```python
# NEW CODE - SECURE
try:
    SECRET_KEY = os.environ['DJANGO_SECRET_KEY']
except KeyError:
    raise RuntimeError(
        "CRITICAL: DJANGO_SECRET_KEY environment variable is not set.\n"
        "This is required for security (session cookies, CSRF tokens, JWT signatures).\n\n"
        "Generate a secure key with:\n"
        "  python -c 'from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())'\n"
    )
```

**Impact:**
- ✅ No fallback value exists anywhere
- ✅ Application crashes immediately if env var missing
- ✅ Forces explicit security configuration
- ✅ Impossible to accidentally run with known key
- ⚠️ Local development now requires setup (see `backend/LOCAL_SETUP.md`)

**Setup for Development:**
```bash
# Generate a key
python -c 'from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())'

# Set it
export DJANGO_SECRET_KEY='your-generated-key-here'

# Or create .env.local file (see .env.local.example)
```

## iOS App Transport Security (ATS)

### Development Build
Add to `ios/Runner/Info.plist`:
```xml
<key>NSAppTransportSecurity</key>
<dict>
    <key>NSAllowsLocalNetworking</key>
    <true/>
</dict>
```

### Production Build
Remove ATS exceptions. iOS will enforce HTTPS automatically.

## Android Network Security

### Development Build
In `android/app/src/main/AndroidManifest.xml`:
```xml
android:usesCleartextTraffic="true"
```

### Production Build
Change to:
```xml
android:usesCleartextTraffic="false"
```
