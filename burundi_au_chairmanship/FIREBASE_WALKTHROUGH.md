## 🐛 Troubleshooting Common Issues

### Issue 1: "Invalid configuration"
**Error:** When enabling Apple provider in Firebase

**Solution:**
- Double-check Team ID: `5UL786DM5B`
- Verify Bundle ID matches in both Firebase and Apple Developer
- Try disabling and re-enabling

### Issue 2: "The operation couldn't be completed"
**Error:** When testing Apple Sign In in app

**Solution:**
- Wait 5-10 minutes after configuring Apple Developer (propagation delay)
- Verify Firebase redirect URI was added to Apple Developer correctly
- Check device is running iOS 13+ or macOS 10.15+
- Restart the app

### Issue 3: "Sign in with Apple is not available"
**Error:** Button doesn't appear or is grayed out

**Solution:**
- iOS version must be 13.0 or higher
- Simulator must be iOS 13.4 or higher
- Entitlements file must be configured (already done ✅)

### Issue 4: User has no name/email
**Error:** User profile shows null name or email

**Solution:**
- **Name:** Apple only provides on first sign-in (store it immediately!)
- **Email:** User chose "Hide My Email" - you'll get relay email instead
- Both are expected behaviors for privacy

### Issue 5: "Unauthorized redirect URI"
**Error:** Sign-in fails with redirect error

**Solution:**
- Verify EXACT redirect URI in Apple Developer:
  ```
  https://b4africa-700f7.firebaseapp.com/__/auth/handler
  ```
- Check for typos, extra spaces, or wrong protocol (https vs http)
- Save changes in Apple Developer and wait 10 minutes

## 🎓 Understanding the Flow

### What Happens When User Signs In:

```
1. User taps Apple button in app
   ↓
2. iOS shows native Apple Sign In sheet
   ↓
3. User authenticates (Face ID/password)
   ↓
4. User chooses to share/hide email
   ↓
5. Apple generates ID token
   ↓
6. Firebase receives and verifies token
   ↓
7. Firebase creates/returns user account
   ↓
8. App receives Firebase ID token
   ↓
9. App sends Firebase ID token to Django backend
   ↓
10. Django backend verifies token with Firebase
    ↓
11. Django creates/updates user profile
    ↓
12. User is logged in! 🎉
```

### Data Flow:
- **Apple provides:** User ID, email (or relay), name (first time only)
- **Firebase stores:** Authentication credentials, UID, provider info
- **Django stores:** User profile, preferences, app-specific data
